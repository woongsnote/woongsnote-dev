## 웹의 구조

웹은 네트워크 상에서 동작하는 클라이언트-서버 구조로 이루어져 있다.
클라이언트는 웹 브라우저를 통해 사용자가 요청한 웹 페이지를 서버에 요청하고,
서버는 해당 페이지를 생성하여 응답합니다. 이 과정에서 웹의 구성 요소와 동작 원리는 다음과 같다.

### 웹 브라우저

    - 웹 브라우저는 사용자가 웹 페이지에 접속할 때 사용되는 소프트웨어.
    - 사용자는 웹 브라우저를 통해 웹 페이지에 접속하고, 웹 페이지의 내용을 볼 수 있다.
    - 브라우저는 웹 페이지를 해석하고, 사용자에게 시각적으로 표현하는 역할을 한다.

### 웹 서버

    - 웹 서버는 클라이언트가 요청한 웹 페이지를 생성하는 소프트웨어.
    - 웹 서버는 클라이언트의 요청을 받아들이고, 요청된 웹 페이지를 생성하여 클라이언트에게 응답한다.
    - 서버는 다양한 프로그래밍 언어로 작성될 수 있으며, 클라이언트의 요청에 따라 필요한 데이터를 처리한다.

### HTTP 프로토콜

    - HTTP 프로토콜은 클라이언트와 서버 간의 통신을 가능하게 해주는 프로토콜.
    - HTTP는 클라이언트의 요청과 서버의 응답을 정의하고, 데이터의 전송 방식을 규정한다.

### HTML, CSS, JavaScript

    - HTML은 웹 페이지의 내용을 기술하기 위한 마크업 언어.
    - CSS는 웹 페이지의 스타일과 레이아웃을 기술하는 스타일 시트 언어.
    - JavaScript는 웹 페이지의 동적인 기능을 구현하기 위한 프로그래밍 언어.

    이 세 가지 요소는 웹 페이지를 구성하는 데 필요한 핵심적인 요소다.

### DNS(Domain Name System)

    - DNS는 도메인 이름과 IP 주소 간의 매핑을 제공하는 시스템입.
    - 사용자가 웹 페이지의 도메인 이름을 입력하면, DNS 서버는 해당 도메인 이름에 대한 IP 주소를 찾아 클라이언트에게 반환한다.
    - 이를 통해 클라이언트는 서버에 접속할 수 있게 된다.

이러한 요소들이 함께 작동하여 사용자는 **웹 브라우저**를 통해 **웹 페이지**에 **접속** 하고, 서버는 해당 **페이지를 생성** 하여 사용자에게 **응답** 한다.
