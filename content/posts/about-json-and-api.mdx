## 1. JSON

> **JavaScript Object Notation**의 약자로, 데이터를 효과적으로 **교환**하고 **저장**하기 위한 경량의 데이터 형식이다.

### JSON의 특징

- 중괄호(`{}`)로 시작하고 끝난다.
- 중괄호 안에는 키(key) 와 값(value)의 쌍으로 이루어진 속성들이 포함된다.
- 키는 문자열로 표현된다.
- 값은 다양한 데이터 타입(문자열, 숫자, 불리언, 배열, 객체 등)이 가능하다.

### JSON 예시 코드

```JSON
{
  "name": "John Doe",
  "age": 25,
  "email": "johndoe@example.com",
  "hobbies": ["reading", "coding", "gaming"],
  "address": {
    "city": "Seoul",
    "country": "South Korea"
  }
}
```

## 2. API와 JSON 활용하기

**API(응용 프로그래밍 인터페이스)** 는 서로 다른 시스템 간에 데이터를 교환하기 위한 방법이다.
웹 애플리케이션에서는 서버에서 제공되는 데이터를 클라이언트(프론트엔드)가 활용하여 동적인 화면 구성 및 다양한 기능을 수행한다.

API는 주로 **JSON 형식**으로 데이터를 클라이언트에게 반환한다. 이러한 JSON 데이터를 활용하여 프론트엔드에서 웹 페이지를 업데이트하거나 다른 서비스와의 데이터 교환 등을 수행할 수 있다.

JSON 데이터를 사용하는 방법은 일반적으로 아래와 같이 진행된다.

1. API 요청: 클라이언트는 서버의 API에 요청을 보낸다. 이 요청은 주로 **HTTP 메소드(GET, POST, PUT, DELETE 등)** 와 함께 API 엔드포인트(URI)를 포함한다.

2. 데이터 수신: 서버는 해당 요청을 처리하고 필요한 데이터를 JSON 형식으로 응답한다. 이 JSON 데이터는 클라이언트로 전송된다.

3. JSON 파싱: 클라이언트는 받은 JSON 데이터를 파싱하여 프로그래밍 언어의 데이터 구조로 변환한다. 대부분의 프로그래밍 언어에서는 JSON을 파싱하기 위한 내장 함수 또는 라이브러리를 제공한다.

4. 데이터 활용: 파싱된 데이터는 클라이언트에서 다양한 용도로 활용될 수 있다. 예를 들어, 웹 페이지의 요소를 업데이트하거나, 다른 API에 데이터를 전송하거나, 데이터를 가공하여 표시하는 등의 작업을 수행할 수 있다.

프론트엔드 개발에서, JSON과 API는 필수적인 도구이다. JSON 형식의 데이터를 잘 다루고 API와의 상호작용을 효과적으로 구현할 수 있다면, 웹 애플리케이션의 성능과 사용자 경험을 향상시킬 수 있다.

이제 JSON 데이터를 사용하기 위해 직접 사용해 본, 두 가지 JavaScript 라이브러리인 Fetch와 Axios에 대해 알아보겠다.

### Fetch

**Fetch API** 는 JavaScript의 내장된 웹 API로, 네트워크 요청을 생성하고 응답을 처리하는 기능을 제공한다.

- Fetch의 특징

  - 간단하고 가벼운 API 요청을 다룰 때 주로 사용된다.
  - Promises를 기반으로 한 비동기 처리 방식을 사용하며, .then()과 .catch()를 이용해 응답과 에러를 처리한다.
  - ES6의 기능으로 지원되기 때문에 추가적인 라이브러리 설치가 필요하지 않다.
  - 최신 브라우저에서는 대부분 지원되지만, 오래된 브라우저에서는 폴리필(polyfill)이 필요할 수 있다.

- Fetch를 사용하여 JSON 데이터를 요청하고 처리하는 방법

```javascript
fetch(url)
  .then((response) => response.json())
  .then((data) => {
    // JSON 데이터 처리
  })
  .catch((error) => {
    // 에러 처리
  });
```

### Axios

**Axios**는 브라우저와 Node.js에서 모두 동작하는 인기 있는 JavaScript HTTP 클라이언트 라이브러리다.

- Axios의 특징

  - 복잡한 HTTP 요청을 처리하고자 할 때 사용된다. 예를 들어, 요청 취소, 인터셉터(interceptor), 데이터 변환 등의 기능을 지원한다.
  - Promises를 사용하는 비동기 처리 방식을 지원하며, .then()과 .catch()뿐만 아니라 .finally()를 통한 종료 처리도 제공한다.
  - 추가적인 기능과 설정을 제공하기 때문에 다양한 환경에서 사용할 수 있다.
  - 외부 라이브러리로서 설치가 필요하며, npm 등을 통해 패키지를 설치해야 한다.

- Axios를 사용하여 JSON 데이터를 요청하고 처리하는 방법

```javascript
axios
  .get(url)
  .then((response) => {
    const data = response.data;
    // JSON 데이터 처리
  })
  .catch((error) => {
    // 에러 처리
  });
```

그러므로, Fetch와 Axios는 각각의 장단점과 사용 사례가 있으므로 프로젝트 요구사항과 개발 환경에 맞게 선택해야 한다.
간단한 API 요청이 필요하거나 최신 브라우저에서 작업하는 경우에는 Fetch를 사용하는 것이 적합하며, 복잡한 기능이나 다양한 환경에서 작업하는 경우에는 Axios를 활용하는 것이 유용할 수 있다.

## 결론

JSON의 개념과 API에서 JSON을 활용하는 방법에 대해 알아보았다.
또한, Fetch와 Axios라는 JavaScript 라이브러리를 사용하여 JSON 데이터를 요청하고 처리하는 방법을 살펴보았다.
JSON과 API를 잘 활용하여 프론트엔드 개발을 진행하면 웹 애플리케이션의 성능과 사용자 경험을 향상시킬 수 있을 것이다.
