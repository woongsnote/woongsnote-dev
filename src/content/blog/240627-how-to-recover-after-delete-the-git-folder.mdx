Git은 개발자들에게 필수적인 버전 관리 도구다. 하지만 가끔 실수로 .git 폴더를 삭제하는 경우가 생길 수 있다. 이런 상황에서 당황하지 않고 대처할 수 있도록, .git 폴더 삭제 후 커밋을 복구하는 방법과 관련 Git 명령어를 자세히 알아본다.

## .git 폴더의 중요성

.git 폴더는 Git 저장소의 핵심이다. 이 폴더에는 모든 커밋 히스토리, 브랜치 정보, 설정 등이 저장되어 있다. 그래서 이 폴더를 삭제하면 프로젝트의 전체 Git 히스토리가 사라질 수 있으니 조심해야 한다.

## 복구 과정과 관련 Git 명령어

.git 폴더를 실수로 삭제했다면, 다음 단계를 따라 복구를 시도할 수 있다:

1. **새로운 Git 저장소 초기화**

   ```bash title="git 초기화"
   git init
   ```

   이 명령어는 현재 디렉토리에 새로운 .git 폴더를 만든다.

2. **원격 저장소 추가**

   ```bash title="git 저장소 추가"
   git remote add origin [원격 저장소 URL]
   ```

   이 명령어로 로컬 저장소와 원격 저장소를 연결한다.

3. **원격 저장소에서 데이터 가져오기**

   ```bash title="원격 저장소 데이터 가져오기"
   git fetch --all
   ```

   이 명령어는 원격 저장소의 모든 브랜치와 태그 정보를 가져온다.

4. **원격 브랜치를 로컬에 적용**

   ```bash 원격 브랜치 적용
   git reset --hard origin/main
   ```

   이 명령어는 로컬의 main 브랜치를 원격의 main 브랜치와 동일한 상태로 만든다.

## 추가 유용한 Git 명령어

- `git status`: 현재 저장소의 상태를 확인할 수 있다.
- `git log`: 커밋 히스토리를 볼 수 있다.
- `git branch`: 브랜치 목록을 보여준다.
- `git checkout -b [브랜치명]`: 새 브랜치를 만들고 전환한다.
- `git merge [브랜치명]`: 현재 브랜치에 다른 브랜치를 병합한다.

## 주의사항과 팁

1. 정기적으로 원격 저장소에 푸시하는 습관을 들이자.
2. 중요한 로컬 변경사항은 항상 백업해둔다.
3. Git 명령어를 사용할 때는 신중해야 한다.

Git을 효과적으로 사용하면 개발 과정이 훨씬 수월해진다. 이 글에서 소개한 명령어들을 잘 익혀두면 Git 사용 중 발생할 수 있는 여러 문제 상황에 대처할 수 있을 것이다.
