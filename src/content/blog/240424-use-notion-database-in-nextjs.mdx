## 개요

최근에 노션 데이터베이스에 저장된 데이터를 기반으로 Next.js 프로젝트를 만들었다.
이 때의 경험을 토대로, 노션과 Next.js에 대해 어느 정도의 경험이 있다고 가정하고 노션 데이터베이스 기반 Next.js 프로젝트를 구현하는 방법을 정리해보려고 한다.

## 구현 순서

전체 과정을 요약하면 아래와 같다.

1. 노션 페이지 생성
2. 데이터베이스 생성
3. API key 발급
4. Next.js 프로젝트 생성
5. 데이터베이스 연결
6. 페이지 렌더링
7. 배포

각 단계별로 살펴보도록 하자.

## 구현

### 1. 노션 페이지 생성

[노션](https://notion.so)에 접속해서 데이터베이스로 사용할 새로운 페이지를 생성한다.

### 2. 데이터베이스 생성

생성된 페이지에서 데이터베이스를 생성한다.

### 3. API Key 발급

API key를 발급받는다.

### 4. Next.js 프로젝트 생성

1. `npx create-next-app@latest` 로 `Next.js` 프로젝트를 생성한다.
2. VsCode로 생성한 프로젝트를 열고, root 폴더에 `.env` 파일을 생성한다.
3. `.env` 파일에 발급받은 API key와 Database 주소를 추가한다.
4. `.gitignore`에 `.env`를 추가한다.
5. 완료되었으면, `npm run dev`로 정상적으로 프로젝트가 실행되는지 확인해본다.
6. 문제없이 실행된다면, 이제 노션 데이터베이스의 데이터를 가져와서 보여주기만 하면 된다.

### 5. 노션 데이터베이스 연결

노션 데이터베이스의 데이터를 가져오기 위해, 아래 패키지를 설치하도록 하자.

- @notionhq/client: ㅎ
- @notion-render/client: ㅎ

```bash title="install packages"
npm install @notionhq/client @notion-render/client
```

### 6. 페이지 렌더링

App router를 사용할 것이므로, app 폴더 안에 posts 폴더를 만들고, 그 안에 page.tsx 파일을 생성한다.

```tsx title="posts/page.tsx"

```

posts 폴더 안에 `[slug]` 폴더를 생성하고, 그 안에 page.tsx 파일을 생성한다. slug 를 기반으로 상세 페이지를 생성하기 위함이다.

```tsx title="[slug]/page.tsx"

```

### 7. 배포

프로젝트가 정상적으로 동작한다면, Vercel을 통해서 배포해보자.
[Vercel](https://vercel.com)에 접속해서, 프로젝트를 `import`하고, 이전에 만들었던 `.env` 파일의 환경 변수를 등록하자.

배포가 완료되면, 생성된 URL로 접속해본다. 오류없이 동작한다면 모든 단계가 완료되었다.
