## 개요

Next js 블로그를 구현할 때, Ghost CMS를 사용해서, 빠르게 블로그를 만드는 방법에 대해 정리해보려고 한다.

해당 방법은 [여기]() 내용을 참고해서 구현했다.

## 구현 순서

구현 방법의 순서를 요약하면 아래와 같다.

1. Ghost Blog 생성
2. Ghost Blog에서 API 테스트를 위한 Post Sample 작성
3. Ghost CMS API key 발급

4. Next.js 프로젝트 생성
5. Next.js 프로젝트에 Ghost CMS API 사용을 위한 라이브러리 설치
6. Next.js 프로젝트에 API 연동
7. Next.js 프로젝트에서 컴포넌트 구현

## 구현

### 1. Ghost Blog 생성

[Ghost 사이트](https://account.ghost.org/signup) 에서 포스트를 저장할 사이트를 생성한다.

※ Ghost에서 제공해주는 템플릿 그대로 사용한다면, 이대로 Blog로 사용할 수도 있다.😊

### 2. Post Sample 작성

Ghost CMS API와 연동해서, Nextjs 프로젝트에서 게시글이 잘 불러오는 지를 테스트하기 위해, 3개 정도의 포스트를 작성한다.

### 3. API key 발급

Ghost CMS API와 연동하기 위한 API key가 필요하다.

API key는 설정 페이지(사이트 왼쪽 하단의 톱니바퀴 아이콘을 클릭하면 들어올 수 있다)에서, Integration에 Custom을 선택해서 생성할 수 있다.

생성된 Custom Integration을 클릭하면, Content API key를 확인할 수 있다. 

### 3. Next.js 프로젝트 생성

```bash
npx create-next-app@latest
```

터미널에서 위 명령어를 실행하면, 아래와 같이 설정할 수 있다.
내가 진행한 설정은 프로젝트 이름은 **next-ghost-blog**로 설정하고, TypeScript, ESLint, Tailwind CSS, App Router를 사용하고, src directory를 사용하지 않도록 설정했다.

※ 설정의 경우, 취향에 따라 다르게 설정할 수도 있다.

```
√ What is your project named? ... next-ghost-blog
√ Would you like to use TypeScript? ...  Yes
√ Would you like to use ESLint? ...  Yes
√ Would you like to use Tailwind CSS? ...  Yes
√ Would you like to use `src/` directory? ... No
√ Would you like to use App Router? (recommended) ... Yes
√ Would you like to customize the default import alias (@/*)? ... No
```

### 4. Next.js 프로젝트에 Ghost CMS API 사용을 위한 라이브러리 설치

```bash
npm install @tryghost/content-api
```

※ TypeScript를 사용한다면, 아래 라이브러리의 추가가 필요하다!

```bash
npm i --save-dev @types/tryghost__content-api
```
